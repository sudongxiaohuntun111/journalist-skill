## 三、测试场景评估（evals/test-prompts.json）

由于无法在本环境中加载 journalist skill 进行实时交互模拟，以下基于 SKILL.md 指令对 3 个测试场景进行**静态推演评估**。

### 场景 1：`"想写点东西"`（典型场景）

| 评估维度 | 预期行为 | 指令覆盖度 | 评价 |
|---------|---------|-----------|------|
| 采访流程自然展开 | 进入阶段 1（破冰定位） | ✅ SKILL.md 第 56–63 行 | 清晰 |
| 搜索能力触发 | 采访中搜索背景 | ❌ 无触发指令 | 未覆盖 |
| 输出文章有悬念开头 | 5 种开头方法 | ✅ 第 208–225 行 | 清晰 |
| 情感弧线 | 3 种弧线型 | ✅ 第 229–263 行 | 清晰 |
| 画面感细节（≥2） | 4 类细节选取标准 | ✅ 第 267–284 行 | 清晰 |
| 结尾有设计 | 4 种结尾类型 | ✅ 第 305–315 行 | 清晰 |
| 无 AI 味 | 自检清单 | ✅ 第 320–328 行 | 有指令但需 AI 自律 |

**结论**：场景 1 的采访流程和写作步骤覆盖较好。**搜索能力完全未覆盖**。

### 场景 2：`"帮我写一篇关于AI客服的文章"`（有话题场景）

#### 建议 1：重写 `scripts/search_helper.py` 为可执行搜索脚本

```python
"""
采访辅助搜索脚本。
对接 web_search 工具，在采访前/中快速获取背景信息。
用法: python3 search_helper.py <关键词>
"""
import sys
import json

def search_web(query):
    """调用 web_search 工具或 API 执行搜索"""
    # 这里应该调用实际的搜索接口
    # 在 Hermes 环境中对接 web_search 工具
    print(json.dumps({
        "action": "search",
        "query": query,
        "source": "journalist-skill"
    }))

if __name__ == "__main__":
    if len(sys.argv) < 2:
        print("用法: python3 search_helper.py <搜索关键词>")
        sys.exit(1)
    search_web(" ".join(sys.argv[1:]))
```

## 六、总结

**搜索能力是最大的结构性缺陷**。SKILL.md 写了一整节"搜索能力"，`scripts/search_helper.py` 也建了文件，但两者在实际执行中都不产生任何效果——搜索从未被触发，脚本从未被调用。这直接导致 CLAUDE.md 中用户反馈的问题"搜索是泛泛的"、"研究做了但没吃进去"。

**采访状态机结构合理但缺少精确性**。5 阶段设计符合新闻采访的认知流程，但切换条件依赖 AI 的"模糊判断"。加上元指令和量化阈值后，可执行性将大幅提升。

**写作工作流是项目中最强的一环**。5 步法有技法、有示例、有自检清单，体现了中西方新闻写作研究的成果。问题在于缺少"从素材到成文"的生成式模板，导致 AI 知道"应该做"但不知道"具体怎么做"。
